CONCEPTOS DE RED

Red informática:

Es un conjunto de dispositivos interconectados que se comunican para compartir recursos e intercambiar datos mediante protocolos de comunicación, utilizando conexiones físicas (cables) o inalámbricas (WiFi).


Red LAN (Local Area Network):

Es un grupo de dispositivos interconectados (ordenadores, impresoras, etc.) en una ubicación geográfica limitada, como un hogar, una oficina o un edificio, que les permite compartir recursos como archivos, impresoras e Internet de forma rápida y segura.


Red WAN (Wide Area Network):

Es una red informática que conecta equipos y redes locales (LAN) en una gran área geográfica, como ciudades, países o continentes, permitiendo la comunicación entre ubicaciones remotas, siendo el ejemplo más grande y conocido la propia Internet.


Internet:

Es una red de computadoras interconectadas a nivel mundial que facilita el intercambio de información, servicios y recursos en tiempo real. Esta red funciona a través de un conjunto de protocolos de comunicación que hace posible la conexión entre dispositivos ubicados en diferentes lugares.


Web y WWW:

Es un sistema de información global de documentos y recursos interconectados (texto, imágenes, videos) que se accede a través de Internet, la red física de computadoras; la WWW funciona sobre Internet, utilizando navegadores para saltar de una página a otra mediante enlaces (hipervínculos), mientras que Internet es la infraestructura subyacente que permite esta comunicación.


Navegador:

Es un programa o aplicación que permite acceder, ver e interactuar con los contenidos de la World Wide Web (Internet), interpretando códigos como HTML para mostrar páginas, videos e imágenes de forma visual y comprensible para el usuario. Ejemplos: Google Chrome, Microsoft Edge, Safari, Opera y Mozilla Firefox.


Protocolo en informática:

Es un conjunto de reglas y estándares que definen cómo los dispositivos se comunican, envían, reciben y procesan datos en una red,asegurando que puedan entenderse entre sí, como un "lenguaje común" para máquinas, especificando formato, sincronización y manejo de errores para una transmisión eficiente y ordenada.


Protocolo HTTP, HTTPS:

HTTP (Protocolo de Transferencia de Hipertexto) es el protocolo básico para la comunicación web, pero envía datos en texto plano, mientras que HTTPS (HTTP Seguro) es su versión mejorada y cifrada, que usa SSL/TLS para encriptar la información, añadiendo un candado en el navegador y protegiendo datos sensibles como contraseñas y datos bancarios.


DNS:

DNS (Domain Name System) es el sistema que traduce nombres de dominio (como google.com) a direcciones IP, permitiendo que navegadores y servidores se encuentren.


Página web estática VS Página web dinámica:

Una página web estática muestra el mismo contenido a todos los usuarios, codificado directamente en HTML y CSS, sin interactuar con bases de datos, mientras que una página dinámica genera contenido personalizado y en tiempo real según el usuario o la hora, usando lenguajes de servidor (como PHP) y bases de datos para interacciones como inicios de sesión, comentarios o compras, ofreciendo mayor flexibilidad a costa de complejidad y recursos. La diferencia clave es la generación de contenido: estático es pre-construido y fijo; dinámico se construye bajo demanda y es personalizable.


CMS (Ventajas, inconvenientes y ejemplos):

Es un software que permite crear, administrar y modificar sitios web sin necesidad de saber programar, facilitando la gestión de textos, imágenes y otros contenidos a través de una interfaz intuitiva, ofreciendo ventajas como facilidad de uso, personalización y ahorro de costes, aunque con posibles inconvenientes como la necesidad de mantenimiento y la sobrecarga de recursos si no se gestiona bien. Ejemplos:WordPress y Drupal.


Proveedor de servicios de internet:

Es una empresa que te conecta a internet, actuando como un portal entre tus dispositivos y el mundo en línea, mediante tecnologías como fibra, DSL o satélite, y ofreciendo servicios adicionales como email, alojamiento web y seguridad. Ejemplos: Movistar y Vodafone.


Dirección IP (Privada y Pública):

Es un identificador numérico único para dispositivos en una red, como el internet, y la diferencia clave es que la IP pública identifica tu red ante el mundo exterior (Internet) y es asignada por tu ISP, mientras que la IP privada es una dirección interna, asignada por tu router, para que los dispositivos (PC, móvil, TV) se comuniquen entre sí dentro de tu red doméstica o de empresa.


Alojamiento web/hosting:

Es el servicio de alquilar un espacio en un servidor para guardar todos los archivos, datos e información de una página web, haciéndola accesible y visible en Internet 24/7. Ejemplos: Arsys y Webempresa.


Dominio:

Es la dirección única y fácil de recordar para un sitio web en Internet (como google.com), que sirve para identificarlo y permitir a los usuarios acceder a él sin necesidad de recordar complejas direcciones IP numéricas (como 192.168.1.1).


Cliente/Servidor:

Es un modelo informático que divide tareas entre un cliente (solicita servicios/datos, interactúa con el usuario) y un servidor (provee servicios, procesa y almacena datos), comunicándose a través de una red para compartir recursos de forma eficiente, centralizando la gestión de datos y lógica de negocio en el servidor para mejorar la seguridad y escalabilidad.


2. DISEÑO DE PÁGINAS WEB: Layout


Uso de DIV en HTML:

Es una etiqueta HTML genérica que funciona como un contenedor de bloque para agrupar otros elementos, sin tener un significado semántico propio, y se usa principalmente para aplicar estilos con CSS o manipularlos con JavaScript, ayudando a estructurar y diseñar una página web, como crear secciones o columnas.


Layout en HTML:

Es la disposición y organización visual de elementos en un espacio, ya sea una página web, un documento, una tienda física o un almacén, con el objetivo de guiar al usuario, optimizar procesos y mejorar la estética y funcionalidad. Define dónde van textos, imágenes, maquinaria, etc., para crear una estructura lógica y eficiente, siendo un plan o esquema fundamental antes de la implementación final.
